Proxmox와 Docker를 이용해 서버를 구축하고 관리함에 따라 가상화 기술에 대한 관심도가 커지게 되었습니다.

이번 글에서는 가상화의 종류는 어떤 것이 존재하는지, 각 가상화의 장단점과 특징에 대해서 이야기 해 보겠습니다.

## 가상화란?

가상화는 물리적인 하드웨어를 논리적으로 분할하여 여러 독립적인 시스템을 구축하기 위한 기술입니다.

가상화 기술을 이용하면 다음과 같은 이점을 누릴 수 있게 됩니다.

1. 리소스의 효율적 사용 : 가상화를 통해 하나의 물리적인 하드웨어에서 여러 독립적인 시스템을 운용 할 수 있게 되므로 기존의 1하드웨어 - 1서비스 체계보다 효율적인 리소스의 사용이 가능해집니다.
2. 시스템 격리 : 가상화는 시스템의 격리를 통해 가상 시스템들이 서로 영향을 끼치거나 받는 것을 방지합니다. 이는 시스템의 보안성과 안정성을 높여줄 수 있습니다.
3. 개발과 테스트 용이성 : 가상 시스템은 쉬운 복제와 생성, 파괴가 가능하고 격리되어 있기 때문에 하나의 가상 시스템을 이용해 실제 서비스 환경과 같은 환경을 구축할 수 있고, 호스트에 관계 없이 일관적인 개발 환경을 제공 할 수 있습니다.

하지만 다음과 같은 단점도 존재합니다.

1. 성능 저하 : 가상화 기술을 이용하면 필연적으로 추가적인 레이어가 필요하게 되므로 성능의 저하가 발생합니다.
2. 호환성 : 하드웨어에 의존적인 어플리케이션들은 가상화 시스템에서 동작이 불가능 할 수 있습니다.
3. 시스템 복잡성 증가 : 가상 머신을 생성하고 관리하는데 드는 비용이 증가할 수 있습니다.
